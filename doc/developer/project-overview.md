# Project Overview

## Vision

Build native desktop and mobile apps with Python, powered by Rust, designed especially for AI, data, automation, education, ERP, and business workflows.

## Product Positioning

PyNative UI should not start as a direct Flutter replacement. The strongest early niche is Python-first AI/data/business software where Python is already the main language.

The framework should feel structurally familiar to Flutter developers, but Pythonic in syntax:

- Python app code defines the UI tree, state, routing, and callbacks.
- Rust handles performance-sensitive runtime work.
- Native platform shells provide lifecycle, permissions, packaging, and platform services.
- Backend mode supports heavy AI/data packages that are unrealistic to bundle on mobile.

## Primary Audience

- Python developers.
- AI engineers.
- Data scientists.
- Automation developers.
- Internal tool builders.
- ERP and business workflow developers.
- Students and educators.

## First Platforms

- Windows desktop.
- Android mobile.

## Later Platforms

- Linux.
- macOS.
- iOS.

## Core Product Principles

- Be honest about mobile Python limitations.
- Use native widgets first for smaller app size and native feel.
- Keep the Python API simple and declarative.
- Make packaging a core feature from the beginning.
- Support backend mode for heavy AI/data workloads.
- Build trust with clear docs, examples, and compatibility tables.

## Best First GitHub Tagline

PyNative UI - Build native AI/data apps with Python, powered by Rust.
