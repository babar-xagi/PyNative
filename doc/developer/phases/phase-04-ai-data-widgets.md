# Phase 4 - AI/Data Widgets

## Goal

Add higher-level widgets and workflows that make PyNative UI valuable for AI, data, automation, and business apps.

## Deliverables

- `Table` widget for business/data display.
- Chart widget or chart integration strategy.
- Chat UI components for AI apps.
- File picker.
- Progress widget for long-running jobs.
- Background job API.
- Demo apps for AI chat, data table, and file analyzer.

## Workstreams

- Virtualized table rendering.
- Sorting, selection, and simple filtering.
- Data binding for rows and columns.
- Chat message list, input area, and streaming response state.
- File picker integration on Windows and Android.
- Background task queue and UI-safe progress updates.

## Acceptance Criteria

- A table can display realistic business data.
- A chat UI can stream or append model output.
- A file picker can select a local file where platform permissions allow it.
- Background tasks do not freeze the UI.
- Demo apps clearly show the AI/data/business niche.

## Exit Gate

Phase 4 is complete when PyNative UI can demonstrate apps that feel meaningfully different from basic GUI examples.

## Enhancement Added

Prioritize accessibility for data widgets early: keyboard navigation, labels, focus states, and readable table interactions.
